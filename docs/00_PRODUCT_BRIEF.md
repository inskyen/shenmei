# 00_PRODUCT_BRIEF：審美者項目簡報

## 1. 我們在做什麼

**審美者** 是一個以影片為入口的審美策展社交產品。

使用者不是單純收藏影片，而是發布一條帶有個人審美判斷的內容：

```text
一支影片 + 一段推薦理由 + 可選小館
```

產品最重要的不是「影片有多熱門」，而是：

> 某個人為什麼選擇它，以及這個選擇如何透露他的審美。

## 2. 雙核心

審美者採用 **video + post 雙核心**。

| 核心 | 作用 | 對應頁面 |
| --- | --- | --- |
| `video` | 影片本體，負責長期沉澱 | `/v/[id]` |
| `post` | 使用者的一次推薦，負責信息流與人格表達 | `/p/[id]` |

首頁與小館信息流展示 `posts`。  
影片頁沉澱一支 `video` 的播放與所有推薦；第一版的討論收斂在各篇 `post` 內。

## 3. 第一版 MVP

第一版只做一個閉環：

```text
註冊 / 登入
→ 發布 B 站影片
→ 填寫推薦理由
→ 可選小館
→ 發布成功後回到大廳最新流
→ 看到自己剛發布的內容
```

這個閉環完成後，產品就從展示原型變成真正的策展社交雛形。

## 4. 已確認規則

- 註冊 / 登入後即可發布。
- 第一版只支援 B 站影片。
- 推薦理由必填。
- 小館由管理員建立。
- 發布時可選小館，不選則進入大廳最新流。
- 發布成功後跳回 `/` 大廳最新流。
- `reactions` 第一版只做 `like`。
- 底部 `探索` 是 `/search`，用來搜尋 `video`。
- 頂部 `小館` 是 `/m`，用來瀏覽小館。
- 私訊與通知拆分為 `/messages` 與 `/notifications`。
- 使用者頁 `/u/[username]` 第一版做極簡版。

## 5. 主要模組

| 模組 | 第一版狀態 | 說明 |
| --- | --- | --- |
| 帳號 / profile | 必做 | 登入、暱稱、頭像、數字 username |
| 影片 video | 必做 | B 站影片本體資料 |
| 策展 post | 必做 | 使用者推薦影片的一次動態 |
| 大廳 feed | 必做 | 全站最新 posts |
| 小館 modules | 必做基礎 | 管理員建立，使用者可投稿 |
| 搜尋 search | 占位 | 未來搜尋 video |
| 喜歡 reactions | 必做 | 第一版只做 like |
| 留言 comments | 必做簡版 | 支持 post / video 目標 |
| 使用者頁 | 必做極簡 | 展示資料與發布列表 |
| 私訊 messages | 占位 | 後續做文字私訊 |
| 通知 notifications | 占位 | 評論、回覆、喜歡等通知 |

## 6. 產品氣質

審美者的氣質是：

```text
安靜展廳感為主
Soul 式輕社交為暗線
小館板塊為骨架
```

它應該柔和、低飽和、有策展感。  
不要做成吵鬧社群，也不要做成普通影片站。

## 7. 第一階段最重要的事

第一階段不是做完整社區，而是把策展動態做出來：

- 使用者能發布一支影片。
- 使用者必須寫推薦理由。
- 內容能進大廳最新流。
- 使用者能點進影片頁與推薦詳情。

一句話：

> 先讓審美者擁有第一條真正由使用者留下的審美痕跡。
