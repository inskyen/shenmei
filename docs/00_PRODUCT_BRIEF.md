# 00_PRODUCT_BRIEF：審美者項目簡報

## 1. 我們在做什麼

**審美者** 是一個以影片為入口的審美採樣社交產品。

使用者不是單純收藏影片，而是發布一條帶有個人審美判斷的內容：

```text
一支影片 + 一段推薦理由 + 可選頻道
```

產品最重要的不是「影片有多熱門」，而是：

> 某個人為什麼選擇它，以及這個選擇如何透露他的審美。

## 2. 雙核心

審美者採用 **video + post 雙核心**。

| 核心 | 作用 | 對應頁面 |
| --- | --- | --- |
| `video` | 影片本體，負責長期沉澱 | `/v/[id]` |
| `post` | 使用者的一次推薦，負責信息流與人格表達 | `/p/[id]` |

首頁與頻道信息流展示 `posts`。
影片頁沉澱一支 `video` 的播放與所有推薦；第一版的討論收斂在各篇 `post` 內。

## 3. 當前階段

截至 2026-07-16，第一版核心 MVP 已完成，可進行小規模種子內測。

目前不是「準備開始施工」，而是：

```text
核心閉環完成
→ 補齊反濫用、治理與運維
→ 種子內測
→ 公開測試
```

## 4. 第一版 MVP

第一版只做一個閉環：

```text
註冊 / 登入
→ 發布 B 站影片
→ 填寫推薦理由
→ 可選頻道
→ 發布成功後回到大廳最新流
→ 看到自己剛發布的內容
```

這個閉環完成後，產品就從展示原型變成真正的採樣社交雛形。

這個閉環已經完成，並已延伸出追蹤、通知、私訊、頻道治理與內容刪除。

## 5. 已確認規則

- 註冊 / 登入後即可發布。
- 第一版只支援 B 站影片。
- 推薦理由必填。
- 頻道由管理員建立。
- 發布時可選頻道，不選則進入大廳最新流。
- 發布成功後跳回 `/` 大廳最新流。
- `reactions` 第一版只做 `like`。
- 首頁右上角搜尋入口是 `/search`，用來搜尋 `video` 與使用者。
- 首頁頂部與底部 `頻道` 會切換至頻道列表；點擊頻道卡片後進入 `/m/[slug]`。
- 私訊與通知拆分為 `/messages` 與 `/notifications`。
- 使用者頁 `/u/[username]` 第一版做極簡版。
- 普通使用者只能向大廳採樣；審美者與超管可以選擇頻道。
- 採樣刪除後保留影片本體；已刪除主留言保留回覆脈絡。

## 7. 產品氣質

審美者的氣質是：

```text
安靜展廳感為主
Soul 式輕社交為暗線
頻道板塊為骨架
```

它應該柔和、低飽和、有採樣感。  
不要做成吵鬧社群，也不要做成普通影片站。

## 8. 下一階段最重要的事

採樣閉環已經完成。下一階段不是繼續堆頁面，而是讓產品可以承受陌生使用者：

- 防止批量註冊、垃圾採樣與私訊騷擾。
- 讓使用者可以檢舉、封鎖並理解社區規則。
- 讓超管可以處理違規內容與帳號。
- 讓錯誤、備份與回滾都有明確入口。

一句話：

> 先讓審美者安全地接住第一批真正的陌生使用者。
