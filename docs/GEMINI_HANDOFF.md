# AI 工程交接：審美者當前狀態

> 更新時間：2026-07-16
> 適用對象：接手本專案的 Gemini、Codex 或其他開發協作者。

## 1. 一句話理解產品

審美者是一個以 Bilibili 影片為入口的審美採樣社交產品，立場是：

> 輔助您改造個性化默認推薦。

使用者發布的是：

```text
一支影片 + 一段推薦理由 + 可選頻道
```

不要把它做成 B站下載工具、熱門影片榜、後台管理系統或普通短影片 App。

## 2. 核心模型

產品採用 `video + post` 雙核心：

- `video`：影片本體與長期沉澱，對應 `/v/[id]`。
- `post`：某個人的一次採樣、推薦理由與討論，對應 `/p/[id]`。

同一支影片可以被多人重複採樣。首頁、頻道與個人頁的主內容單位都是 `post`。

## 3. 當前階段

```text
核心 MVP 已完成
→ 可進行小規模種子內測
```

## 5. 角色與權限

| 身份 | 權限 |
| --- | --- |
| 訪客 | 瀏覽公開內容；受限操作提示登入 |
| `member` | 向大廳採樣、互動、追蹤、私訊 |
| `aesthete` | 擁有 member 權限，可選擇頻道投稿並顯示徽章 |
| `super_admin` | 可管理頻道、移出內容、刪除違規採樣與留言 |

角色由 Supabase 手動調整，前端禁止使用者自行提權。

## 6. 已完成功能

### 帳號與 profile

- Email / Password 註冊登入。
- 六位 Email OTP。
- 自動建立 profile 與唯一數字審美號。
- 暱稱、頭像、簡介、審美標籤與私訊權限設定。

### 發現首頁

- 「追蹤／最新／頻道」。
- 左右滑動切換。
- 最新流分頁載入、返回位置恢復與頁面預載。
- 沉浸播放、喜歡、留言數與未讀入口。

### 採樣與影片

- B站連結／BVID 解析、影片去重與採樣發布。
- `/v/[id]` 影片沉澱頁。
- `/p/[id]` 採樣詳情、留言、回覆與喜歡。
- 詳情播放器隨捲動等距縮小。
- 採樣與留言軟刪除。

### 頻道

- 頻道列表與詳情。
- 超管建立、排序與關閉頻道。
- 普通使用者只能向大廳採樣；審美者與超管可選頻道。

### 人與社交

- 使用者頁、採樣懶載入、追蹤與追蹤流。
- 使用者與影片搜索。
- 喜歡、留言、回覆與追蹤通知。
- 一對一文字私訊、Realtime 與未讀數。

### App 化体验

- 固定底部導航。
- 頁面預載與記憶體緩存。
- 骨架載入與返回狀態保留。
- 品牌蝴蝶 favicon、分享圖與主畫面圖標。

## 7. 不得破壞的工程邊界

- 不把 `post` 合併回 `video`。
- 不刪除或重建現有 Supabase 表。
- 不改路由語義。
- 不繞過 RLS 或把 service role key 放進前端。
- 不為視覺重構破壞緩存、預載、Realtime、鑑權與返回位置。
- 所有使用者介面文字維持台灣繁體中文。
- 現有 UI 方向是克制、黑白、內容優先、略帶冷感；不要加入大面積漸變、玻璃擬態或廉價科技感。

## 8. 當前最高優先級

公開測試前先補治理地基，不再優先增加展示頁：

1. 註冊、採樣、留言與私訊限流。
2. 檢舉、封鎖、封禁與超管治理隊列。
3. `/api/bilibili` 域名白名單、超時、快取與限流。
4. CSP 與其他安全響應頭。
5. 隱私政策、使用者協議、社區規範與聯絡入口。
6. 錯誤監控、備份、回滾、自動化測試與 CI。

推薦算法應排在治理能力之後，避免更快放大垃圾內容。

## 9. 資料庫協作方式

- migration 位於 `supabase/migrations/`。
- 目前由使用者在 Supabase SQL Editor 手動執行。
- 寫入新 migration 後，必須把完整 SQL 交給使用者並等待執行結果。
- 不得假設檔案存在就代表遠端已套用。
- 涉及 RLS、trigger、function 時，必須提供實際權限驗證步驟。

## 10. 驗證方式

```bash

```
