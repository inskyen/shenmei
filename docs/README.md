# 審美者文檔索引

> 文檔基線：2026-07-16
> 當前階段：核心 MVP 完成，準備種子內測與公開測試前治理。

審美者是一個以影片為入口的審美採樣社交產品，採用 **video + post 雙核心**：

- `video`：影片本體，負責內容沉澱，對應 `/v/[id]`。
- `post`：使用者的一次採樣，負責信息流、人格表達與討論，對應 `/p/[id]`。

## 當前里程碑

核心產品閉環已成立：

```text
註冊／登入
→ 發布 B站影片與推薦理由
→ 選擇頻道或進入最新大廳
→ 產生採樣詳情與影片沉澱頁
→ 喜歡、留言、追蹤、通知與私訊
```

目前可以進行小規模種子內測；公開測試前仍需完成：

- 註冊、採樣、留言與私訊的反濫用限流。
- 檢舉、封鎖、封禁與超管治理入口。
- B站代理接口安全加固。
- 隱私政策、使用者協議、社區規範與侵權處理入口。
- 錯誤監控、備份、回滾與正式部署檢查單。
- 自動化測試與 CI 基線。

## 已確認產品決策

- 第一版只支援 B站影片，但資料結構保留多平台可能性。
- 普通使用者可向大廳採樣；審美者可選擇頻道；超管可管理頻道與內容。
- `reactions` 第一版只做 `like`。
- 首頁為「追蹤／最新／頻道」，底部為「發現／頻道／採樣／私訊／我的」。
- `/v/[id]` 沉澱影片本體，`/p/[id]` 承載每次採樣及其討論。
- 採樣軟刪除後保留影片本體；已刪除主留言保留回覆脈絡。
- 私訊與通知分離，私訊使用 Supabase Realtime。
