# 03_ROUTES：審美者頁面與路由文檔

## 1. 路由設計原則

審美者的頁面應圍繞四個核心對象展開：

```text
video   影片本體
post    策展動態
module  頻道
profile 使用者
```

首頁和頻道頁展示的主資料應該是 `posts`。
影片主頁 `/v/[id]` 則以 `video` 為中心，沉澱播放、影片資訊、所有推薦與影片評論。

## 2. 路由總覽

| 路由 | 頁面 | MVP | 說明 |
| --- | --- | --- | --- |
| `/` | 大廳 / 最新流 | 是 | 全站最新策展動態 |
| `/submit` | 發佈策展 | 是 | 發布 B 站影片 + 推薦理由 |
| `/v/[id]` | 影片主頁 | 是 | 單支影片的沉澱頁 |
| `/p/[id]` | 策展動態詳情 | 是 | 單篇 post 詳情 |
| `/m` | 頻道列表 | 是 | 所有頻道入口 |
| `/m/[slug]` | 頻道頁 | 是 | 某個頻道下的 posts |
| `/u/[username]` | 使用者頁 | 是 | 極簡使用者資料與發布列表 |
| `/login` | 登入 / 註冊 | 是 | 帳號入口 |
| `/messages` | 訊息列表 | 占位 | 私訊入口 |
| `/messages/[id]` | 私訊詳情 | 否 | 後續實作 |
| `/search` | 搜尋 | 占位 | 搜尋 video |
| `/notifications` | 通知列表 | 占位 | 評論、回覆、喜歡等通知 |
| `/settings` | 設定 | 否 | 後續實作 |

## 3. `/` 大廳

### 定位

首頁大廳是全站入口，預設展示全站最新策展動態。

它不是某個頻道，也不是影片列表。

### 頂部導航

```text
追蹤 / 最新 / 頻道
```

第一版：

- `最新` 為預設 active。
- `追蹤` 顯示目前使用者追蹤之人的公開策展動態。
- `頻道` 在首頁內切換至頻道列表，點擊卡片後進入 `/m/[slug]`。

### 底部導航

```text
發現 / 頻道 / 採樣 / 私訊 / 我的
```

MVP 行為：

- `大廳` → `/`
- `探索` → `/search`
- `發佈` → `/submit`
- `訊息` → `/messages`
- `我的` → `/u/[current_username]`，未登入則 `/login`

### 主要資料

首頁應查：

```text
posts
profiles
videos
modules
like_count
comment_count
```

### 卡片內容

每張策展卡片應展示：

- 策展人暱稱與頭像。
- 發布時間。
- 推薦理由 `post.note`。
- 影片封面。
- 影片標題。
- UP 主 / 作者。
- 所屬頻道。
- 喜歡數。
- 留言數。

### 互動

- 點擊封面：開啟沉浸播放，或後續進入 `/v/[id]`。
- 點擊影片標題：進入 `/v/[id]`。
- 點擊推薦文字或卡片：進入 `/p/[id]`。
- 點擊策展人：進入 `/u/[username]`。
- 點擊頻道：進入 `/m/[slug]`。

### 空狀態

```text
還沒有策展動態。成為第一個留下審美痕跡的人。
```

## 4. `/submit` 發佈策展

### 定位

讓使用者發布一條策展動態。

發布行為像 Soul 瞬間，但內容是影片策展。

### 訪問權限

- 未登入：導向 `/login`。
- 已登入：可發布。

### 表單欄位

必填：

- B 站連結或 BVID。
- 推薦理由。

選填：

- 頻道，可多選。
- 未選頻道時，預設發到大廳最新流。

後續可自動解析：

- 影片標題。
- 封面。
- UP 主。

若第一版無法自動解析，可先允許手動填入。

### 校驗

- B 站連結或 BVID 必填。
- 推薦理由必填。
- 推薦理由建議最少 10 個字。

### 寫入流程

```text
檢查登入狀態
→ 解析 BVID
→ 查 videos 是否已存在
→ 不存在則建立 videos
→ 建立 posts
→ 若選擇頻道，建立 post_modules
→ 發布成功後跳轉 `/`
→ 大廳最新流立即展示剛發布的策展動態
```

### 成功提示

```text
已發布，這支影片有了新的審美註解。
```

### 發布後跳轉

發布成功後回到 `/` 大廳最新流，而不是直接進入 `/p/[id]` 或 `/v/[id]`。

原因：

- 讓使用者立即看到自己剛發布的內容出現在公共大廳。
- 形成「剛發就被收錄」的即時反饋。
- 更接近 Soul 瞬間動態發布後回到信息流的感覺。

## 5. `/v/[id]` 影片主頁

### 定位

以影片為中心的沉澱頁，接近 B 站影片頁的角色。

它展示的不是某一個人的推薦，而是一支影片在審美者裡累積的所有推薦與討論。

### 主要資料

```text
video
posts for this video
profiles for those posts
comments where target_type = video
modules related through posts
```

### 頁面內容

- 影片播放器。
- 影片標題。
- UP 主 / 作者。
- 原始來源連結。
- 推薦過這支影片的人。
- 所有推薦理由列表。
- 影片公共留言區。
- 相關頻道。

### 互動

- 播放影片。
- 對影片留言。
- 點擊某條推薦進入 `/p/[id]`。
- 點擊推薦者進入 `/u/[username]`。
- 點擊頻道進入 `/m/[slug]`。

### 空狀態

如果影片還沒有公共留言：

```text
這支影片還沒有留下公共回聲。
```

## 6. `/p/[id]` 策展動態詳情

### 定位

展示單篇策展動態的完整內容。

它關注的是某個人的一次推薦，而不是影片的全部沉澱。

### 主要資料

```text
post
profile
video
modules
comments
reactions
```

### 頁面內容

- 影片播放器。
- 影片標題。
- UP 主 / 作者。
- 策展人。
- 推薦理由。
- 所屬頻道。
- 喜歡按鈕。
- 留言區。

### 互動

- 喜歡 / 取消喜歡。
- 發表針對這條推薦的留言。
- 點擊影片標題進入 `/v/[id]`。
- 點擊策展人進入 `/u/[username]`。
- 點擊頻道進入 `/m/[slug]`。

### 留言區空狀態

```text
還沒有人留言。也許你可以寫下第一句回聲。
```

## 7. `/m` 頻道列表

### 定位

展示所有頻道入口。

頻道是審美者的板塊 / 主題空間，不是臨時房間。

### 主要資料

```text
modules
每個 module 的 post_count
每個 module 的最新 posts 預覽，可後續做
```

### 頻道卡片

每張卡片展示：

- 頻道名稱。
- 頻道簡介。
- 封面或主色。
- 策展動態數。
- 最新更新時間。

### 頻道類型

類型頻道：

- 影像
- 音樂
- 動畫
- 設計

審美氛圍頻道：

- 藍色時刻
- 孤獨夏天
- 深夜放映
- 城市雨聲

產品應更突出審美氛圍頻道。

### 空狀態

```text
頻道正在整理展牆。
```

## 8. `/m/[slug]` 頻道頁

### 定位

展示某個頻道下的策展動態。

### 主要資料

```text
module
posts
profiles
videos
like_count
comment_count
```

### 頁面內容

- 頻道名稱。
- 頻道簡介。
- 頻道封面 / 主色。
- 頻道內最新策展動態。
- 發佈到此頻道的入口。

### 子導航

第一版可先只做：

```text
最新
```

後續可加入：

```text
精選 / 熱門
```

### 權限

- 所有人可讀 active 頻道。
- 管理員可建立 / 編輯頻道。
- 使用者可投稿到頻道。

## 9. `/u/[username]` 使用者頁

### 定位

展示某個使用者的審美檔案。

### MVP 狀態

第一版必做極簡版。

原因：

- 每條策展動態都需要能點進推薦者。
- 底部 `我的` 需要有落點。
- 私訊與追蹤未來都依賴使用者頁。

### 主要資料

```text
profile
posts
follower_count
following_count
```

### 頁面內容

- 頭像。
- 暱稱。
- 簡介。
- 發布過的策展動態。

第一版不做：

- 審美標籤。
- 追蹤按鈕。
- 私訊按鈕。
- 完整資料編輯。

### 空狀態

```text
這裡還沒有留下策展痕跡。
```

## 10. `/login` 登入 / 註冊

### 定位

帳號入口。

### 功能

- Email 登入。
- Email 註冊。
- 註冊後建立 profile。
- 註冊後自動生成一串數字型 `username`，後續可在個人設定中修改。

### 文案

- `歡迎回來`
- `申請成為策展人`
- `信箱`
- `密碼`
- `登入`
- `註冊`

### 待修

目前登入頁應改為使用環境變數初始化 Supabase，不應硬編碼 Supabase URL 與 key。

## 11. `/search` 搜尋

### 定位

首頁右上角搜尋入口對應此頁，用來搜尋 `video` 與使用者。

它不是頻道列表；頻道可由首頁頁籤或底部導航進入。

### MVP 狀態

第一版可先占位。

### 未來功能

- 搜尋 B 站影片資料。
- 搜尋站內已收錄 videos。
- 點擊影片進入 `/v/[id]`。
- 若影片未收錄，可引導使用者發布策展。

### 占位文案

```text
搜尋正在準備中。之後可以在這裡找到影片。
```

## 16. 現有頁面對應

```text

```

```text
pages/submit.js
```

## 17. 第一階段路由施工順序

建議順序：

1. 保留 `/`，但資料源從 `videos` 過渡到 `posts`。
2. 修 `/login` Supabase 初始化。
3. 新增 `/submit` 發布策展。
4. 新增 `/v/[id]` 影片主頁。
5. 新增 `/p/[id]` 策展動態詳情頁。
6. 新增 `/m` 頻道列表。
7. 新增 `/m/[slug]` 頻道頁。
8. 新增 `/search` 占位。
9. 新增 `/messages` 占位。
10. 新增 `/notifications` 占位。
11. 新增 `/u/[username]` 基礎個人頁。
